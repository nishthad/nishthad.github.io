The Idea

FamAlert is a concept imagined to work as a two-part product, a device & a mobile phone application. The idea was to have a device, ideally worn by an elderly family member, to which the mobile application would serve as an interface to keep you informed about your loved ones.
FamAlert is a prototype that was done as a Human-Computer-Interaction practicum. The process involved participatory design, user interviews, cognitive walkthroughs, and low-fidelity protoypes that were translated into the final design.

Conceptualization   |   Low and High Fidelity Prototypes + Design Spec   |   Usability Study Analysis


Key Considerations

- Simple interface for all ages, levels of experience
- Intuitive presentation of data and analysis
- Quick glances, and constant presence in case of need



 
 

Making the pieces fit

Having done the participatory design sessions, key functionality and features started to emerge. With a large set of features spanning from location tracking, to sleep analysis, the challenge was to design an interface that delivers all this information in a simple, easy-to-understand manner.



 
 

Participatory Design Session Results

 


In the interview sessions, the particpants were asked about their daily hassles with caregiving, their level of interest in the app, and any difficulties they currently face. Several features were talked about by all participants, for example, alerts, periodic calls, and sleep tracking. They were then asked if there was an app centered around this idea, what they would imagine it to look like. Some of the participants got very intricate with their vision, and the sketches can be seen above.

 
 

Location & Detail Screens

 


Strong inspiration was taken from the user sketches to design screens. The detail and location screens are highly focused on the data they are conveying, which is the graph, and map, respectively.

 
 

Home & Pairing Screens

 


The home screen of the app was designed to give a quick glance of all members that have been added to the app. An overlay for members that require attention was an added layer to ensure that it does not miss the users' attention. The pairing screen used the wi-fi icon to convey that the device is being searched for.

 
 

Alert & Data View

 


The alert screen reflects the features from the user sketches, with options to call, description of the problem, and added features such as remind later, and dismiss. The users wanted a portable data view to be able to share with doctors, which was placed behind the hamburger list menu.

 
 

Overview Screen

 


The overview screen aims to provide a quick summary of the wearer's status. Each list item represents a metric that the user chooses to track, and can be reordered by priority. The screen further offers options to call, set a reminder, or buzz the wearer's band, if the user were to notice anything unsual on this screen.

 
 

Usability Study

 


After the high-fidelity prototypes were created, a usability study was done, where users were given certain tasks to accomplish. Their responses were analyzed, and any difficulties they encountered in completing these tasks were noted.

 
 

Tutorial Mode

 


Lastly, a tutorial mode was included, to be displayed when the app was first downloaded and used. This acknowledged some of the issues that the users ran into during the usability study, as well as some features that appeared to be hidden, as they emerged in the design critique session.
